## Overview

The `make_net` family of functions creates neural network architectures with different configurations optimized for specific accuracy targets on CIFAR-10. Each variant (93, 94, 95, 96) represents the target accuracy percentage.

All networks use:
- Whitening convolution as the first layer
- Three convolutional blocks with pooling
- GELU activation functions
- FP16 mixed precision training
- Channels-last memory format for efficiency

## make_net93

Creates a network optimized for 93% accuracy with minimal computational cost.

```python
def make_net93(
    widths={'block1': 64, 'block2': 128, 'block3': 128},
    batchnorm_momentum=0.6
)
```

### Parameters

<ParamField path="widths" type="dict" default="{'block1': 64, 'block2': 128, 'block3': 128}">
  Dictionary specifying the number of channels for each convolutional block
</ParamField>

<ParamField path="batchnorm_momentum" type="float" default="0.6">
  Momentum parameter for batch normalization layers
</ParamField>

### Returns

A PyTorch `nn.Sequential` model configured with:
- Block widths: 64, 128, 128 channels
- FP16 precision
- Channels-last memory format
- Frozen whitening layer weights

### Architecture

- Whitening convolution (3 → 24 channels, 2x2 kernel)
- ConvGroup 1: 24 → 64 channels
- ConvGroup 2: 64 → 128 channels
- ConvGroup 3: 128 → 128 channels
- Global pooling and linear classifier

### Example

```python
import airbench

# Create 93% accuracy network
model = airbench.make_net93()

# Custom widths
model = airbench.make_net93(
    widths={'block1': 32, 'block2': 64, 'block3': 64},
    batchnorm_momentum=0.7
)
```

## make_net96

Creates a network optimized for 96% accuracy with residual connections.

```python
def make_net96()
```

### Parameters

This function takes no parameters. Network configuration is determined by internal hyperparameters.

### Returns

A PyTorch `nn.Sequential` model configured with:
- Block widths: 128, 384, 512 channels
- Residual connections in each ConvGroup
- FP16 precision
- Channels-last memory format
- Frozen whitening layer weights

### Architecture

- Whitening convolution (3 → 24 channels, 2x2 kernel)
- ConvGroup 1: 24 → 128 channels (3 convs with residual)
- ConvGroup 2: 128 → 384 channels (3 convs with residual)
- ConvGroup 3: 384 → 512 channels (3 convs with residual)
- Global pooling and linear classifier

### Key Differences

- Uses 3 convolutions per block (vs 2 in other variants)
- Includes residual connections
- No configurable parameters (uses fixed hyperparameters)
- Deeper architecture for higher accuracy

### Example

```python
import airbench

# Create 96% accuracy network
model = airbench.make_net96()

# Use with training
model = airbench.train96(make_net=airbench.make_net96)
```

## Common Features

All networks include:

- **Whitening layer**: Learned data preprocessing as first layer
- **Mixed precision**: FP16 computation with FP32 batch norm
- **Memory optimization**: Channels-last format for better performance
- **GPU-ready**: Automatically moved to CUDA device
- **Frozen preprocessing**: Whitening weights are not trained
